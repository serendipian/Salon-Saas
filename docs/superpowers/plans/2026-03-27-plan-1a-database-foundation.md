# Plan 1A: Database Foundation — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Set up Supabase local development, create all 20 database tables, 3 views, all RLS policies, all Postgres functions/triggers, all indexes, and seed data — producing a fully functional, secure database layer with zero frontend changes.

**Architecture:** Supabase CLI for local dev. Migrations split into logical files for reviewability. Every table gets RLS enabled from creation. Session-variable-based RLS using `set_config`/`current_setting`. Audit logging via triggers. Sensitive fields encrypted via pgcrypto.

**Tech Stack:** Supabase CLI, PostgreSQL 15, pgcrypto, btree_gist, PL/pgSQL

---

## File Structure

```
supabase/
  config.toml                                    # Supabase CLI config (auto-generated, we customize auth settings)
  seed.sql                                       # Global seed data (plans table) + dev test data
  migrations/
    00001_extensions_and_helpers.sql              # Extensions + utility functions (updated_at trigger, session helpers)
    00002_identity_tables.sql                     # profiles, salons, plans, salon_memberships, invitations
    00003_business_tables_core.sql                # service_categories, services, service_variants, product_categories, products, suppliers
    00004_people_tables.sql                       # staff_members, clients
    00005_scheduling_tables.sql                   # appointments (with exclusion constraint)
    00006_financial_tables.sql                    # transactions, transaction_items, transaction_payments, expense_categories, expenses, recurring_expenses
    00007_audit_log.sql                           # audit_log table + trigger function + attach triggers to all business tables
    00008_rls_policies.sql                        # RLS enable + all policies for all tables
    00009_views.sql                               # client_stats, appointment_details, ledger_entries
    00010_indexes.sql                             # All composite and specialized indexes
    00011_functions_identity.sql                  # create_salon, accept_invitation, transfer_ownership, revoke_membership, protect_last_owner trigger
    00012_functions_business.sql                  # create_transaction, book_appointment, gdpr_delete_client
    00013_functions_reporting.sql                 # get_dashboard_stats, get_client_stats, get_staff_performance, check_slot_availability
.env.local                                       # VITE_SUPABASE_URL + VITE_SUPABASE_ANON_KEY (local)
```

Each migration file is self-contained. Dependencies flow top-down (later migrations may reference earlier tables but never the reverse).

```sql
-- ============================================================
-- Migration 00001: Extensions and Helper Functions
-- ============================================================

-- Required for exclusion constraints (appointment double-booking prevention)
CREATE EXTENSION IF NOT EXISTS btree_gist;

-- Required for sensitive field encryption (base_salary, iban, ssn)
CREATE EXTENSION IF NOT EXISTS pgcrypto;

-- ============================================================
-- Trigger function: auto-update updated_at on every UPDATE
-- ============================================================
CREATE OR REPLACE FUNCTION update_updated_at()
RETURNS trigger AS $$
BEGIN
  NEW.updated_at = now();
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

-- ============================================================
-- Session context functions for RLS
-- ============================================================

-- Called once per request by the Supabase client wrapper.
-- Sets both active salon and user role in session variables.
-- SECURITY INVOKER: runs as the calling user (no privilege escalation needed).
CREATE OR REPLACE FUNCTION set_session_context(p_salon_id uuid, p_user_role text)
RETURNS void AS $$
BEGIN
  PERFORM set_config('app.active_salon_id', p_salon_id::text, true);
  PERFORM set_config('app.user_role', p_user_role, true);
END;
$$ LANGUAGE plpgsql SECURITY INVOKER;

-- Read the active salon for the current request.
-- SECURITY DEFINER: RLS policies call this — must work regardless of caller's RLS context.
-- STABLE: does not modify data, safe for RLS evaluation.
CREATE OR REPLACE FUNCTION get_active_salon()
RETURNS uuid AS $$
  SELECT nullif(current_setting('app.active_salon_id', true), '')::uuid;
$$ LANGUAGE sql STABLE SECURITY DEFINER;

-- Read the current user's role for the current request.
-- SECURITY DEFINER: same reasoning as get_active_salon().
CREATE OR REPLACE FUNCTION get_user_role()
RETURNS text AS $$
  SELECT nullif(current_setting('app.user_role', true), '');
$$ LANGUAGE sql STABLE SECURITY DEFINER;
```

```sql
-- ============================================================
-- Migration 00002: Identity & Access Tables
-- profiles, plans, salons, salon_memberships, invitations
-- ============================================================

-- ---- profiles ----
-- Global user identity. Created automatically by trigger on auth.users insert.
-- No salon_id — this is a global table.
CREATE TABLE profiles (
  id UUID PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
  email TEXT NOT NULL UNIQUE,
  first_name TEXT,
  last_name TEXT,
  avatar_url TEXT,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TRIGGER profiles_updated_at
  BEFORE UPDATE ON profiles
  FOR EACH ROW EXECUTE FUNCTION update_updated_at();

-- Auto-create profile when a new user signs up
CREATE OR REPLACE FUNCTION create_profile_on_signup()
RETURNS trigger AS $$
BEGIN
  INSERT INTO public.profiles (id, email, first_name, last_name)
  VALUES (
    NEW.id,
    NEW.email,
    COALESCE(NEW.raw_user_meta_data->>'first_name', ''),
    COALESCE(NEW.raw_user_meta_data->>'last_name', '')
  );
  RETURN NEW;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

CREATE TRIGGER on_auth_user_created
  AFTER INSERT ON auth.users
  FOR EACH ROW EXECUTE FUNCTION create_profile_on_signup();

-- ---- plans ----
-- Subscription plan definitions. Stripe-ready, manually managed for V1.
-- No salon_id — global reference table.
CREATE TABLE plans (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL,
  max_staff INTEGER,         -- NULL = unlimited
  max_clients INTEGER,       -- NULL = unlimited
  max_products INTEGER,      -- NULL = unlimited
  features JSONB DEFAULT '{}',
  price_monthly NUMERIC(10,2),
  price_yearly NUMERIC(10,2),
  stripe_price_id_monthly TEXT,
  stripe_price_id_yearly TEXT,
  active BOOLEAN NOT NULL DEFAULT true
);

-- ---- salons ----
-- One row per salon tenant.
CREATE TABLE salons (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL,
  slug TEXT UNIQUE,
  address TEXT,
  phone TEXT,
  email TEXT,
  website TEXT,
  logo_url TEXT,
  currency TEXT NOT NULL DEFAULT 'EUR',
  vat_rate NUMERIC(5,2) DEFAULT 20,
  timezone TEXT NOT NULL DEFAULT 'Europe/Paris',
  schedule JSONB,            -- Weekly opening hours (WorkSchedule structure)
  plan_id UUID REFERENCES plans(id),
  subscription_tier TEXT NOT NULL DEFAULT 'trial'
    CHECK (subscription_tier IN ('trial', 'free', 'pro', 'enterprise')),
  trial_ends_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  deleted_at TIMESTAMPTZ
);

CREATE TRIGGER salons_updated_at
  BEFORE UPDATE ON salons
  FOR EACH ROW EXECUTE FUNCTION update_updated_at();

-- ---- salon_memberships ----
-- Links users to salons with roles. One user can belong to multiple salons.
CREATE TABLE salon_memberships (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  salon_id UUID NOT NULL REFERENCES salons(id) ON DELETE CASCADE,
  profile_id UUID NOT NULL REFERENCES profiles(id) ON DELETE CASCADE,
  role TEXT NOT NULL CHECK (role IN ('owner', 'manager', 'stylist', 'receptionist')),
  invited_by UUID REFERENCES profiles(id),
  invited_at TIMESTAMPTZ,
  accepted_at TIMESTAMPTZ,
  status TEXT NOT NULL DEFAULT 'active'
    CHECK (status IN ('pending', 'active', 'suspended')),
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  deleted_at TIMESTAMPTZ,
  UNIQUE (salon_id, profile_id)
);

CREATE TRIGGER memberships_updated_at
  BEFORE UPDATE ON salon_memberships
  FOR EACH ROW EXECUTE FUNCTION update_updated_at();

-- ---- invitations ----
-- Pending invitations to join a salon.
CREATE TABLE invitations (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  salon_id UUID NOT NULL REFERENCES salons(id) ON DELETE CASCADE,
  email TEXT NOT NULL,
  role TEXT NOT NULL CHECK (role IN ('manager', 'stylist', 'receptionist')),
  token TEXT NOT NULL UNIQUE,
  invited_by UUID NOT NULL REFERENCES profiles(id),
  expires_at TIMESTAMPTZ NOT NULL DEFAULT (now() + interval '7 days'),
  accepted_at TIMESTAMPTZ,
  email_sent_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  UNIQUE (salon_id, email)
);
```

```sql
-- ============================================================
-- Migration 00005: Scheduling Tables
-- appointments (with double-booking exclusion constraint)
-- ============================================================

CREATE TABLE appointments (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  salon_id UUID NOT NULL REFERENCES salons(id) ON DELETE CASCADE,
  client_id UUID REFERENCES clients(id),
  service_id UUID REFERENCES services(id),
  service_variant_id UUID REFERENCES service_variants(id),
  staff_id UUID REFERENCES staff_members(id),
  date TIMESTAMPTZ NOT NULL,
  duration_minutes INTEGER NOT NULL,
  status TEXT NOT NULL DEFAULT 'SCHEDULED'
    CHECK (status IN ('SCHEDULED', 'COMPLETED', 'CANCELLED', 'NO_SHOW')),
  price NUMERIC(10,2) NOT NULL,     -- Snapshotted at booking time
  notes TEXT,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  created_by UUID REFERENCES profiles(id),
  updated_by UUID REFERENCES profiles(id),
  deleted_at TIMESTAMPTZ
);

CREATE TRIGGER appointments_updated_at
  BEFORE UPDATE ON appointments
  FOR EACH ROW EXECUTE FUNCTION update_updated_at();

-- Double-booking prevention via exclusion constraint.
-- Requires btree_gist extension (installed in migration 00001).
-- Prevents two non-cancelled appointments from overlapping for the same staff member.
ALTER TABLE appointments
  ADD CONSTRAINT no_double_booking
  EXCLUDE USING gist (
    staff_id WITH =,
    tstzrange(date, date + (duration_minutes || ' minutes')::interval) WITH &&
  )
  WHERE (status NOT IN ('CANCELLED'));
```

```sql
CREATE TABLE audit_log (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  salon_id UUID,
  table_name TEXT NOT NULL,
  record_id UUID NOT NULL,
  action TEXT NOT NULL CHECK (action IN ('INSERT', 'UPDATE', 'DELETE')),
  old_data JSONB,
  new_data JSONB,
  performed_by UUID REFERENCES profiles(id),
  performed_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- Generic audit trigger function.
-- Reads salon_id from the row (if it exists) and user from auth.uid().
-- SECURITY DEFINER: needs to insert into audit_log regardless of RLS context.
CREATE OR REPLACE FUNCTION audit_trigger()
RETURNS trigger AS $$
DECLARE
  v_salon_id UUID;
  v_record_id UUID;
  v_old JSONB;
  v_new JSONB;
BEGIN
  -- Extract salon_id from the row (all business tables have it)
  IF TG_OP = 'DELETE' THEN
    v_salon_id := OLD.salon_id;
    v_record_id := OLD.id;
    v_old := to_jsonb(OLD);
    v_new := NULL;
  ELSIF TG_OP = 'INSERT' THEN
    v_salon_id := NEW.salon_id;
    v_record_id := NEW.id;
    v_old := NULL;
    v_new := to_jsonb(NEW);
  ELSE  -- UPDATE
    v_salon_id := NEW.salon_id;
    v_record_id := NEW.id;
    v_old := to_jsonb(OLD);
    v_new := to_jsonb(NEW);
  END IF;

  INSERT INTO audit_log (salon_id, table_name, record_id, action, old_data, new_data, performed_by)
  VALUES (v_salon_id, TG_TABLE_NAME, v_record_id, TG_OP, v_old, v_new, auth.uid());

  IF TG_OP = 'DELETE' THEN
    RETURN OLD;
  END IF;
  RETURN NEW;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

-- Attach audit triggers to all business tables
-- (Identity tables: salons, salon_memberships are included because they contain business-critical data)
DO $$
DECLARE
  tbl TEXT;
BEGIN
  FOR tbl IN SELECT unnest(ARRAY[
    'salons',
    'salon_memberships',
    'clients',
    'service_categories',
    'services',
    'service_variants',
    'product_categories',
    'products',
    'suppliers',
    'staff_members',
    'appointments',
    'transactions',
    'transaction_items',
    'transaction_payments',
    'expense_categories',
    'expenses',
    'recurring_expenses'
  ])
  LOOP
    EXECUTE format(
      'CREATE TRIGGER audit_%I AFTER INSERT OR UPDATE OR DELETE ON %I FOR EACH ROW EXECUTE FUNCTION audit_trigger();',
      tbl, tbl
    );
  END LOOP;
END;
$$;
```

```sql
-- client_stats, appointment_details, ledger_entries
-- ============================================================

-- ---- client_stats ----
-- Computes total_visits, total_spent, last_visit_date from transactions.
-- Replaces the denormalized fields on the old Client type.
CREATE OR REPLACE VIEW client_stats AS
SELECT
  c.id AS client_id,
  c.salon_id,
  COUNT(DISTINCT t.id) AS total_visits,
  COALESCE(SUM(t.total), 0) AS total_spent,
  MAX(t.date)::date AS last_visit_date
FROM clients c
LEFT JOIN transactions t ON t.client_id = c.id AND t.salon_id = c.salon_id
WHERE c.deleted_at IS NULL
GROUP BY c.id, c.salon_id;

-- ---- appointment_details ----
-- Joins appointments with client, staff, service, variant names.
-- Replaces the denormalized clientName, staffName, serviceName columns.
CREATE OR REPLACE VIEW appointment_details AS
SELECT
  a.id,
  a.salon_id,
  a.client_id,
  c.first_name || ' ' || c.last_name AS client_name,
  a.service_id,
  s.name AS service_name,
  a.service_variant_id,
  sv.name AS variant_name,
  a.staff_id,
  sm.first_name || ' ' || sm.last_name AS staff_name,

LEFT JOIN services s ON s.id = a.service_id
LEFT JOIN service_variants sv ON sv.id = a.service_variant_id
LEFT JOIN staff_members sm ON sm.id = a.staff_id
```

```sql
    'transaction_count', COUNT(t.id),

    'appointment_count', (
      SELECT COUNT(*) FROM appointments a
      WHERE a.salon_id = p_salon_id
        AND a.date BETWEEN p_date_from AND p_date_to
        AND a.status != 'CANCELLED'
        AND a.deleted_at IS NULL
    ),
    'new_clients', (
      SELECT COUNT(*) FROM clients c
      WHERE c.salon_id = p_salon_id
        AND c.created_at BETWEEN p_date_from AND p_date_to
        AND c.deleted_at IS NULL
    ),
    'top_services', (
      SELECT COALESCE(jsonb_agg(row_to_json(top)), '[]'::jsonb)
      FROM (
        SELECT ti.name, SUM(ti.quantity) AS count, SUM(ti.price * ti.quantity) AS revenue
        FROM transaction_items ti
        INNER JOIN transactions tx ON tx.id = ti.transaction_id
        WHERE tx.salon_id = p_salon_id
          AND tx.date BETWEEN p_date_from AND p_date_to
          AND ti.type = 'SERVICE'
        GROUP BY ti.name
        ORDER BY revenue DESC
        LIMIT 5
      ) top
    ),
    'revenue_by_day', (
      SELECT COALESCE(jsonb_agg(row_to_json(daily) ORDER BY daily.day), '[]'::jsonb)
      FROM (
        SELECT date_trunc('day', t2.date)::date AS day, SUM(t2.total) AS revenue
        FROM transactions t2
        WHERE t2.salon_id = p_salon_id
          AND t2.date BETWEEN p_date_from AND p_date_to
        GROUP BY date_trunc('day', t2.date)::date
      ) daily
    )
  ) INTO v_result
  FROM transactions t
  WHERE t.salon_id = p_salon_id
    AND t.date BETWEEN p_date_from AND p_date_to;

  RETURN v_result;
END;
$$ LANGUAGE plpgsql SECURITY INVOKER;

-- ---- get_client_stats ----
-- Computed visits/spending/last visit. Replaces denormalized fields.
-- SECURITY INVOKER: respects RLS.
CREATE OR REPLACE FUNCTION get_client_stats(p_client_id UUID)
RETURNS JSONB AS $$
DECLARE
  v_result JSONB;
BEGIN
  SELECT jsonb_build_object(
    'total_visits', COUNT(DISTINCT t.id),
    'total_spent', COALESCE(SUM(t.total), 0),
    'last_visit_date', MAX(t.date)::date
  ) INTO v_result
  FROM transactions t
  WHERE t.client_id = p_client_id;

  RETURN v_result;
END;
$$ LANGUAGE plpgsql SECURITY INVOKER;

-- ---- get_staff_performance ----
-- Revenue per stylist, commission calculation.
-- SECURITY INVOKER: respects RLS.
CREATE OR REPLACE FUNCTION get_staff_performance(
  p_salon_id UUID,
  p_staff_id UUID,
  p_date_from TIMESTAMPTZ,
  p_date_to TIMESTAMPTZ
)
RETURNS JSONB AS $$
DECLARE
  v_result JSONB;
  v_commission_rate NUMERIC(5,2);
BEGIN
  -- Get the staff member's commission rate
  SELECT commission_rate INTO v_commission_rate
  FROM staff_members
  WHERE id = p_staff_id AND salon_id = p_salon_id AND deleted_at IS NULL;

  SELECT jsonb_build_object(
    'appointment_count', (
      SELECT COUNT(*) FROM appointments a
      WHERE a.staff_id = p_staff_id
        AND a.salon_id = p_salon_id
        AND a.date BETWEEN p_date_from AND p_date_to
        AND a.status = 'COMPLETED'
        AND a.deleted_at IS NULL
    ),
    'total_revenue', (
      SELECT COALESCE(SUM(a.price), 0) FROM appointments a
      WHERE a.staff_id = p_staff_id
        AND a.salon_id = p_salon_id
        AND a.date BETWEEN p_date_from AND p_date_to
        AND a.status = 'COMPLETED'
        AND a.deleted_at IS NULL
    ),
    'commission_rate', COALESCE(v_commission_rate, 0),
    'commission_earned', (
      SELECT COALESCE(SUM(a.price) * COALESCE(v_commission_rate, 0) / 100, 0) FROM appointments a
      WHERE a.staff_id = p_staff_id
        AND a.salon_id = p_salon_id
        AND a.date BETWEEN p_date_from AND p_date_to
        AND a.status = 'COMPLETED'
        AND a.deleted_at IS NULL
    ),
    'cancelled_count', (
      SELECT COUNT(*) FROM appointments a
      WHERE a.staff_id = p_staff_id
        AND a.salon_id = p_salon_id
        AND a.date BETWEEN p_date_from AND p_date_to
        AND a.status = 'CANCELLED'
        AND a.deleted_at IS NULL
    ),
    'no_show_count', (
      SELECT COUNT(*) FROM appointments a
      WHERE a.staff_id = p_staff_id
        AND a.salon_id = p_salon_id
        AND a.date BETWEEN p_date_from AND p_date_to
        AND a.status = 'NO_SHOW'
        AND a.deleted_at IS NULL
    )
  ) INTO v_result;

  RETURN v_result;
END;
$$ LANGUAGE plpgsql SECURITY INVOKER;

-- ---- check_slot_availability ----
-- Advisory availability check for UI. NOT authoritative — book_appointment is the real guard.
-- SECURITY INVOKER: respects RLS.
CREATE OR REPLACE FUNCTION check_slot_availability(
  p_staff_id UUID,
  p_date TIMESTAMPTZ,
  p_duration_minutes INTEGER
)
RETURNS BOOLEAN AS $$
BEGIN
  RETURN NOT EXISTS (
    SELECT 1 FROM appointments
    WHERE staff_id = p_staff_id
      AND status NOT IN ('CANCELLED')
      AND deleted_at IS NULL
      AND tstzrange(date, date + (duration_minutes || ' minutes')::interval) &&
          tstzrange(p_date, p_date + (p_duration_minutes || ' minutes')::interval)
  );
END;
$$ LANGUAGE plpgsql SECURITY INVOKER;
```
