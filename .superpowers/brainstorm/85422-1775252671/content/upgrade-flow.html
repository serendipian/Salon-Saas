Design Section 2 — Upgrade flow + hard block modal

What happens when a user upgrades, and when they hit a plan limit

Upgrade flow (3 steps)
1
Clic sur "S'abonner →"
Depuis la page Abonnement dans Réglages
2
Stripe Checkout (hosted)
Redirection vers la page Stripe sécurisée. Vous ne touchez jamais aux données de carte.
🔒 checkout.stripe.com/pay/cs_live_...
3
Stripe webhook → Edge Function
Stripe notifie votre app. L'Edge Function met à jour subscription_tier en base.
✓
Retour app → succès
L'utilisateur voit un toast "Abonnement Pro activé !" et le plan se met à jour en temps réel.
✓ Abonnement Pro activé !
Hard block modal (plan limit hit)
Exemple : salon Free essaie d'ajouter un 3ème membre d'équipe (limite = 2)
🔒
